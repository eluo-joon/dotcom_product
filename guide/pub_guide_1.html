Common Rule

1. 기본 사항

  • 별도 형상관리 툴 부재로 CMS 및 로컬 작업을 진행하므로 항시 백업 후 작업.
  • 적응형 사이트로 웹 / 모바일 화면 각각 작업 필요.
    • URL 각각 분리, 단순 브라우저 리사이징으로 디바이스 전환 불가
    • Web: https://product.kt.com/wDic/index.do?CateCode=6003
    • Mobile: https://m.product.kt.com/mDic/index.do?CateCode=6003

2. 네이밍 규칙

  • Class 및 ID 명 하이픈( - ) 사용.
  • 이미지명 언더바( _ ) 사용.
  • KT닷컴 전체 페이지 공통 적용사항

2.

  • 네이밍 기본 원칙
    • 페이지 공통화 요소 또는 고유 페이지의 컨테이너에만 id 사용
    • 그 외 모든 요소에는 class 사용
  • BEM(Block Element Modifier) 모델 적용
    • 클래스 네이밍은 BEM 방식을 따르되, 필요 시 ID도 사용 가능
    • JavaScript 라이브러리 적용 시 BEM 모델 사용 예외 가능 (예: swiper)
    • 고유 블럭화가 가능할 경우 상위 요소명을 생략 가능
  • 각 페이지에 해당하는 고유한 클래스명 사용
    • 페이지별 스타일 적용이 필요할 경우 HTML 파일명의 클래스를 추가하여 사용
    • 예시: story 카테고리 > 회사소개 페이지 → .container.about_us

3. Image 규칙

  • 이미지 파일 네이밍 규칙
    • 영문 소문자, 숫자, 언더바(_) 사용 (예: icon_logo.svg)
    • 아이콘은 기본적으로 SVG 사용
    • 이름은 영문 대문자 및 숫자로 시작할 수 없음
  • 저장 위치 규칙
    • 메인 페이지 이미지 → main 폴더
    • 서브 페이지 이미지 → sub 폴더
  • 서브 페이지 이미지명 규칙
    • mo(모바일)_주메뉴ID_섹션_종류_속성(필요 시)
    • 예시: voc_head_bg.png (PC) / mo_voc_head_bg.png (모바일)

4. Html 파일명 규칙

  • 파일명 규칙 개요
    • 초기에 **IA 메뉴명(영문)**을 기준으로 국문과 영문 파일명을 동일하게 작성
    • 최종 파일명은 실제 IA 메뉴명과 일치하지 않을 수 있음

5. 주석 규칙

  • 수정 시 주석
    <!-- 2024-10-01 수정 // --> <!-- // 2024-10-01 수정 -->
  • 섹션 구분 주석
    <!-- S :: 영역 설명 // --> <!-- // E :: 영역 설명 -->
  • 개발 요청 역역 주석
    <!-- S :: 개발 요청 - 영역 설명 // --> <!-- // E :: 개발 요청 - 영역 설명 -->

6. CSS 분리 및 구조

  • CSS 구조: common, layout 스타일
  • 라이브러리 스타일 파일은 default.css에 모두 반영되어 있습니다.

6. 폴더 구조

root/ │── [assets] │ ├── [css] │ ├── [font] │ ├── [images] │ ├── [js] │ ├── [video] │── [html] │── [en]

7. 작업 시 유의사항

  • CSS 관련 주의 사항
    • 라이브러리 스타일은 default.css에 포함
    • CSS로 스타일을 최대한 구현하고, 자바스크립트 사용을 최소화
